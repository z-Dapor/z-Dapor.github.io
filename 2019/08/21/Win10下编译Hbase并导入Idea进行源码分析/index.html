Win10下编译Hbase并导入Idea进行源码分析

发表于 2019-08-21 | 浏览 次

环境：Hbase 2.1.3，Maven 3.6.0 ，Idea2017.1 内存：8G (内存最好不要低于8G，不然感觉会炸！Idea运行内存推荐最小1G)

0.下载Hbase源码 如hbase-2.1.3 (注意版本 注意版本 注意版本！)

https://hbase.apache.org/downloads.html

一：编译源码

1.安装Cygwin

https://blog.csdn.net/u012778714/article/details/74012013

2.在Cygwin中进入解压后的Hbase-2.1.3文件，执行 mvn clean package -DskipTests

3.编译成功后 将源码导入Idea

问题：导入idea后会存在报错：报file size exceeds configured limit错误

解决：修改intellij IDE安装目录下的bin/idea.properties, 将其中的 idea.max.intellisense.filesize=2500 ；改成大一些，比如 idea.max.intellisense.filesize=99999 ；然后重启问题解决。

二：配置启动参数

三：启动

点击debug启动 看报错。

我这里遇到了编译问题。client下 与 protocol-shaded 下存在相同的class—RPCProtos。报了类重复 错误。

解决：我删除了client下的RPCProtos.java

重新执行debug 此时 会报多次RPCProtos,class 不存在。进入 对应 的类，将 protocol-shaded 下的RPCProtos导入即可。反复执行 直到编译成功。
